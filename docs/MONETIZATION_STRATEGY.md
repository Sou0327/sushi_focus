# Sushi Focus マネタイズ戦略

## 戦略概要

**モデル**: Open Core（パターンA）+ Enterprise（パターンC）の組み合わせ

- **無料版**: MIT License、完全オープンソース
- **個人Pro版**: ライセンスキー制御（月額課金）
- **Enterprise版**: チーム管理機能（ユーザー単価課金）

### ライセンス制御
```typescript
// daemon/src/license/checker.ts
export interface License {
  userId: string;
  plan: 'free' | 'pro' | 'enterprise';
  expiresAt: number;
  features: string[];
}

// 機能制御例
export class TaskManager {
  async addTask(task: Task) {
    const license = await checkLicense();

    const limits = {
      free: 1,
      pro: 10,
      enterprise: Infinity
    };

    if (this.tasks.length >= limits[license.plan]) {
      throw new LicenseError(
        `${license.plan}版は${limits[license.plan]}タスクまでです`
      );
    }

    this.tasks.push(task);
  }
}
```

## 決済プラットフォーム

### 採用: Paddle（推奨）

**理由**:
- Merchant of Record（販売代行）モデル
- 特定商取引法の住所公開が不要
- 消費税・VAT自動処理
- グローバル対応（150カ国以上）

**手数料**: 5% + 決済手数料

**代替案**: Stripe + バーチャルオフィス（売上規模拡大後）

## リスクと対策

### リスク
1. ユーザー獲得が進まない
2. Pro版への転換率が低い
3. Paddle審査落ち
4. 競合の模倣

### 対策
1. Product Hunt / Hacker Newsでの露出
2. Pro機能の「仕事で必須」な価値設計
3. 事前にプロダクト品質を担保
4. 特許・商標は不要（先行者利益を活かす）
